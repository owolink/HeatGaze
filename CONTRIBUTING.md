# Руководство по внесению изменений

Спасибо за интерес к HeatGaze! Мы приветствуем вклад от сообщества. Этот документ описывает процесс внесения изменений в проект.

## Как внести свой вклад

1. Fork репозиторий
2. Создайте ветку для ваших изменений (`git checkout -b feature/amazing-feature`)
3. Зафиксируйте ваши изменения (`git commit -m 'Add some amazing feature'`)
4. Отправьте изменения в ваш fork (`git push origin feature/amazing-feature`)
5. Откройте Pull Request

## Стандарты кода

- Используйте отступ в 2 пробела
- Следуйте существующему стилю кода
- Добавляйте комментарии к сложным участкам кода
- Обновляйте документацию при необходимости

## Сообщение об ошибках

При создании issue, пожалуйста:
1. Опишите проблему подробно
2. Укажите шаги для воспроизведения
3. Добавьте скриншоты, если это поможет
4. Укажите версию браузера и операционной системы

## Разработка

1. Установите зависимости:
```bash
npm install
```

2. Запустите тесты:
```bash
npm test
```

3. Запустите линтер:
```bash
npm run lint
```

## Соглашение о лицензии

Внося изменения, вы соглашаетесь с тем, что ваш код будет распространяться под той же лицензией, что и проект (MIT).
